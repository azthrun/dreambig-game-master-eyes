## Rush & Reaction v1.2.0 Implementation Plan (Number Flash, No Leaderboard)

### Summary
Implement a new game mode, **Number Flash**, alongside existing Speed Tiles without changing Speed Tiles behavior.  
Number Flash adds a second-level menu (`4` to `8` numbers), a flash-and-hide memory round, answer submission, result feedback, and quick replay for the same mode.

Locked decisions from requirement refinement:
- Adjacent-repeat rule: **no adjacent duplicates at all** (`11` invalid).
- Submit availability: **enabled for any non-empty input** (length mismatch allowed; mismatch is evaluated as incorrect).

### Scope and Success Criteria
1. Main menu first level shows `Speed Tiles` and `Number Flash`.
2. Clicking `Number Flash` opens second-level menu with `4` to `8` numbers and `Back`.
3. Number Flash round flow:
   - Empty display box (red background),
   - hidden pre-generated sequence,
   - random wait `< 5s`,
   - sequence visible for `300ms`,
   - hidden again,
   - player types input and submits.
4. Result behavior:
   - exact sequence + order => success message,
   - otherwise failure message + correct sequence shown,
   - `Play Again` always shown for same mode restart.
5. Number generation constraints enforced for every round:
   - digits `0..9`,
   - no identical adjacent digits,
   - no ascending/descending consecutive triples.
6. No Number Flash leaderboard, no Supabase interaction.

### Architecture and State Changes

### 1. Mode Modeling Strategy
Keep current reducer-driven architecture, and extend `GameState` into two gameplay tracks:

- Existing Speed Tiles states (unchanged)
- New Number Flash states (new)

Recommended approach:
- Add a top-level `menu` state that tracks selected game and menu depth.
- Add Number Flash-specific phases to avoid overloading tile-game states.

### 2. `src/game/types.ts` Changes (Public Interfaces)
Add/modify these types:

1. `GameMode = 'speed-tiles' | 'number-flash'`
2. `NumberFlashLength = 4 | 5 | 6 | 7 | 8`
3. Menu model:
   - `MenuView = 'root' | 'speed-tiles' | 'number-flash'`
   - `MenuState` includes `view`
4. New Number Flash gameplay states:
   - `NumberFlashWaitingState` (`phase: 'number_flash_waiting'`, `length`, `sequence`, `delayMs`)
   - `NumberFlashRevealedState` (`phase: 'number_flash_revealed'`, `length`, `sequence`)
   - `NumberFlashInputState` (`phase: 'number_flash_input'`, `length`, `sequence`, `answer`)
   - `NumberFlashResultState` (`phase: 'number_flash_result'`, `length`, `sequence`, `answer`, `isCorrect`)
5. Extend `GameAction` with Number Flash events:
   - `OPEN_GAME_MENU` (`mode`)
   - `BACK_TO_ROOT_MENU`
   - `START_NUMBER_FLASH` (`length`)
   - `NUMBER_FLASH_DELAY_ELAPSED`
   - `NUMBER_FLASH_HIDE_ELAPSED`
   - `SET_NUMBER_FLASH_ANSWER` (`answer`)
   - `SUBMIT_NUMBER_FLASH_ANSWER`
   - `PLAY_NUMBER_FLASH_AGAIN`
   - `RETURN_TO_MENU` (reuse existing for global back)

### 3. `src/game/constants.ts` Additions
Add constants:

- `NUMBER_FLASH_LENGTHS = [4,5,6,7,8]`
- `NUMBER_FLASH_MIN_DELAY_MS = 500`
- `NUMBER_FLASH_MAX_DELAY_MS = 4800`
- `NUMBER_FLASH_REVEAL_MS = 300`

Keep as centralized config for future tuning.

### 4. `src/game/utils.ts` Additions
Add pure helpers:

1. `generateNumberFlashSequence(length: NumberFlashLength): string`
   - Uses digits `0..9`
   - Reject/retry generation until constraints pass
2. `hasAdjacentDuplicate(sequence: string): boolean`
3. `hasConsecutiveTriple(sequence: string): boolean`
   - Checks ascending (`n, n+1, n+2`) and descending (`n, n-1, n-2`)
4. `randomIntInclusive(min: number, max: number): number`
5. `sanitizeNumericInput(value: string): string`
   - Keep digits only (`0-9`) for answer input

### 5. `src/game/reducer.ts` Changes
Add transitions for Number Flash:

1. Menu transitions:
   - root -> number-flash submenu
   - submenu back -> root
2. Start round:
   - generate sequence immediately
   - compute random delay
   - enter `number_flash_waiting`
3. Delay elapsed:
   - waiting -> revealed
4. Reveal elapsed:
   - revealed -> input
5. Input updates:
   - store sanitized answer
6. Submit:
   - compare `answer === sequence`
   - transition to result with `isCorrect`
7. Play again:
   - regenerate new sequence for same length
   - loop back to waiting state
8. Return/back:
   - returns to root menu (same semantics as existing `RETURN_TO_MENU`)

### 6. `src/app/App.tsx` Effect Wiring
Add timer effects for new phases:

1. On `number_flash_waiting`:
   - `setTimeout(delayMs)` -> dispatch `NUMBER_FLASH_DELAY_ELAPSED`
2. On `number_flash_revealed`:
   - `setTimeout(NUMBER_FLASH_REVEAL_MS)` -> dispatch `NUMBER_FLASH_HIDE_ELAPSED`

Keep existing countdown/timer/error-flash effects for Speed Tiles unchanged.

### Validation and Edge Cases
1. Non-digit input is stripped in reducer/component input handler.
2. Empty submit blocked (button disabled).
3. Short/long input allowed (per locked decision), evaluated as incorrect on submit.
4. Timers are always cleared on phase change/unmount.
5. Rapid clicking of `Play Again` cannot create overlapping timeouts.
6. Generated sequence always satisfies constraints before round starts.

### Assumptions and Defaults
1. Existing Speed Tiles leaderboard/features remain unchanged for now.
2. Number Flash has no countdown; it starts directly with hidden wait + flash.
3. Number Flash input allows any non-empty length on submit, with exact-match comparison only.
4. Sequence constraint conflict resolved by user decision: no adjacent duplicates.
5. Accessibility baseline includes keyboard submit and readable result text; advanced ARIA announcements can be added if needed but are not release-blocking unless explicitly required.
