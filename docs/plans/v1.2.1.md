## Speed Tiles Rapid-Tap Responsiveness Plan (Keep Sound + Visual Feedback)

### Summary
Keep both tap feedback features (tile visual effect and tap sound), but move them to a lower-priority, adaptive path so they never degrade rapid tapping performance.  
Primary gameplay input (`onPress`) remains immediate and deterministic; feedback may be selectively skipped during burst tapping.

### Confirmed Product Decisions
1. Keep both features enabled (do not remove sound/effect).
2. Use adaptive skipping during bursts to protect responsiveness.
3. Optimize for mid-range mobile as minimum target.

---

### Goals and Success Criteria
1. Rapid tapping should feel as responsive as if no feedback existed.
2. Correct/wrong tile processing timing remains unchanged.
3. Visual effect still appears during normal pace taps.
4. Tap sound still plays during normal pace taps.
5. During very fast burst taps, some feedback events may be skipped, but gameplay input is never skipped.

---

### Implementation Approach

### 1. Input-first tap pipeline in `GameTile`
File: `src/components/speed-tiles/GameTile.tsx`

- Reorder handler so `onPress(tile.id)` is always first and synchronous.
- Move feedback work to a separate helper called after `onPress`.
- Do not block or await audio/effect work in the tap handler.
- Keep tap behavior on `onClick` (no gameplay logic changes).

Planned handler flow:
1. `onPress(tile.id)` immediately.
2. `emitTapFeedback()` (best-effort, adaptive, non-blocking).

### 2. Adaptive feedback gating (burst protection)
File: `src/components/speed-tiles/GameTile.tsx` (or small helper in `src/game/audio.ts` if shared)

Introduce lightweight per-tile refs:
- `lastVisualFeedbackAtRef`
- `lastSoundFeedbackAtRef`

Add cooldown constants:
- `TAP_FEEDBACK_VISUAL_MIN_GAP_MS` (recommended: `28`)
- `TAP_FEEDBACK_SOUND_MIN_GAP_MS` (recommended: `40`)

Behavior:
- If current tap occurs before min gap since last feedback, skip that feedback channel.
- Gameplay input still always executes.
- Visual and sound can skip independently (adaptive policy).

### 3. Keep visual effect but avoid React-state churn
File: `src/components/speed-tiles/GameTile.tsx`, `src/styles.css`

Current visual feedback uses `useState(showTapHint)` and re-renders each tap.  
Replace with DOM-class toggling via `buttonRef` to minimize render overhead:

- Add `buttonRef` to tile button.
- In `emitVisualFeedback`, imperatively:
  - remove `tile-tap-hint`
  - force reflow (`void el.offsetWidth`) only when needed to restart animation
  - add `tile-tap-hint`
- Let CSS animation auto-finish and remove class on `animationend` (or keep timeout fallback).

CSS update:
- Convert `.tile-tap-hint` into a short animation class (instead of state-driven timed class lifecycle).
- Duration recommendation: `70â€“90ms` (brief but visible).

### 4. Keep sound but reduce audio-path overhead
File: `src/game/audio.ts`

Preserve sound playback, but optimize for burst taps:
- Keep a single initialized `AudioContext`.
- Add adaptive sound gap gate.
- Minimize per-tap work:
  - short envelope + short-lived oscillator remains acceptable,
  - avoid extra object churn where possible.
- Add hard safety cap for sound overlap (polyphony cap recommended: `2`) to prevent audio queue pile-up on fast taps.

### 5. Constants and configuration
File: `src/game/constants.ts`

Add:
- `TAP_FEEDBACK_VISUAL_MIN_GAP_MS`
- `TAP_FEEDBACK_SOUND_MIN_GAP_MS`
- `TAP_HINT_ANIMATION_MS` (if CSS duration is centralized)

Keep existing:
- `TILE_TAP_HINT_MS` can be retired if animationend-based cleanup is adopted; otherwise repurpose consistently.

### 6. No gameplay logic changes
Files intentionally unchanged in behavior:
- `src/game/reducer.ts` (tile correctness logic)
- `src/components/speed-tiles/GameBoard.tsx`
- Wrong-click penalties, board flash, win/loss, timer behavior.
