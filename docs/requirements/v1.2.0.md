# Rush & Reaction Requirements

## Version

1.2.0

## Game Name

Rush & Reaction

## New Feature: Number Flash

### Overview

Add a new reflex game mode called **Number Flash**. The player briefly sees a generated number sequence, then types it back in exact order.

### Main Menu (Level 1)

Add one new button on the first-level main menu:

1. **Number Flash**

### Number Flash Menu (Level 2)

When player clicks **Number Flash**, open a second-level menu with:

1. **4 Numbers**
2. **5 Numbers**
3. **6 Numbers**
4. **7 Numbers**
5. **8 Numbers**

This second-level menu must also include:

- **Back** button (returns to first-level menu)
- Proper title and subtitle for this mode
  - Title: `Number Flash`
  - Subtitle: `Memorize the flash, type the exact sequence`

### Gameplay Flow

For each selected mode (4 to 8 digits):

1. Show a sequence display box sized for the selected length.
2. At game start, display box is empty with some red background.
3. Render generated sequence behind the scene.
4. Wait random delay `< 5 seconds` (recommended range: `0.5s` to `4.8s`).
5. Display the sequence in the display box with some blue background and white fonts.
5. Hide the sequence after `300ms` (configurable in the constant file).
6. Player enters the remembered sequence into an input field.
7. On submit:
   - If exact match (value and order), show success message.
   - If mismatch, show failure message and reveal correct sequence.
8. Always show **Play Again** button to restart same mode immediately.

### Sequence Generation Rules

Each digit is randomly generated from `0-9`.

Generation constraints:

1. No 2 identical adjacent digits (forbid runs like `77`, `55`, `22`).
2. No 3 consecutive stepping digits anywhere in sequence:
   - Forbid ascending triples such as `123`, `456`, `789`
   - Forbid descending triples such as `321`, `654`, `210`

Example valid sequences:
- `1120`
- `9071`
- `5540`

Example invalid sequences:
- `1114` (three same adjacent)
- `6785` (contains ascending triple `678`)
- `4329` (contains descending triple `432`)

### UI/UX Requirements

- Mobile and desktop responsive behavior consistent with existing game.
- Input field should use numeric keypad on mobile.
- Submit should always be available as long as input box contains value, even if the length is not matching the answer length.
- Support quick retry via **Play Again** without returning to previous menus.

### Out of Scope for v1.2.0

- No player profile/account system
- No multiplayer/real-time features

## Refinements and Recommendations

Before implementation, confirm these product choices:

1. **Input format**
   - Recommended: digit-only, no spaces/symbols, exact length required.
2. **Attempts per round**
   - Recommended: single submit per round, then show result + Play Again.
3. **Accessibility**
   - Announce result messages and ensure keyboard-only flow works.

## Technical Requirements

- Preserve existing app architecture and route/state patterns.
- Reuse shared menu/modal/button styles where possible.
- Add unit tests for sequence generation constraints.
- Add interaction tests for Number Flash flow:
  - Mode selection
  - Flash and hide timing
  - Correct/incorrect submit behavior
  - Play Again restarting same mode
